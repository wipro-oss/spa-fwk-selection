SPA Framework Selection

A simple SPA framework selection helper

Client/User Environment
  • Interent vs. Intranet (Uncontrolled vs. Controlled | Trusted vs Untrusted)
  • Channel Coverage (Desktop, Smartphone, Tablets)
  • Connectivity (Speed of network connectivity)
User Interface
  1. Standard UI elements vs. Custom UI elements
  2. Data Push to clients
  3. Handle large number of data elements
  4. Validation support
  5. Same UI metaphor across channels vs. Mobile First
Server Environment
  1. Green field vs. Brown field
  2. Service Interfaces (XML vs JSON, HTTP methods)
  3. Isomorphic capability (Capability exists to execute code on server when needed, reuse models/validations)
Developer Support
  1. Knowledge of working with Open Source Communities
  2. Level of familiarity with Javascript
  3. Tooling requirements
Application Needs
  1. Size of possible codebase
  2. Complexity of data entities invovled (model richness and attribute types)
  3. SEO support
  4. Modularity (functional modules, lazy load modules)
  5. Embeddable / partial usage (portal or SSO)
  6. Offline needs
